Remarks:

  • My list extends madcock's list for the SF2000, which also applies to the GB300 v2. He checked all sets known to the stock FBAlpha (called MAME by the interface).
  • My list is about Neo Geo sets that you could “make the stock emulator load”. GB300+SF2000 Tool 2.0-alpha automatically does this when you try to add any supported set from this list.
  • All sets in my list are non-merged FBNeo sets.
  • Major commercial games that are now fully working for the first time on frog: mslug5fd, samsh5fd and ss5spfd
  • This list omits sets in madcock's list that I couldn't improve. There were only two that I could successfully improve, mslug5nd and bangbedp. The latter is the fully-decrypted clone (not a prototype!) of the already-supported bandbead.
  • Even though this list means to extend madcock's list and uses the same ratings, I admit that there are differences in how we rated.
    • Other than madcock, I wouldn't call gpilots and gpilotsh slow. But changing the status here would claim an improvement that isn't there, so I omitted them.
    • On the other hand, I would call mslug3 and mslug5 slow in some places, even in the base version. To be in line with madcock, I didn't mark them as slow.
    • There are some hacks of mslug2, mslug3 and mslug5 that are much worse than the base games due to massive enemy spawns and weapon effects, and these have been marked as slow in this table.
    • madcock tagged quite some games as non-English. I however only used that tag for the two non-English games that you cannot enjoy if you don't speak their language: quizdaisk and quizkofk. I believe that you can perfectly enjoy smashing your enemies' heads even if the text is in Japanese and in this case shouldn't care.
  • Individual set remarks:
    • amazoneo and proneon are rotated. It only makes sense to play these when you're on the SF2000 and use a wireless gamepad.
    • bjourneyh is hardly playable due to missing tiles. I counted it as unplayable. The base game is supported.
    • cphd freezes when you insert a coin, but is free to play.
    • horekidb3 is very hard to find. Do not insert a coin before it tells you to.
    • kof97bs is so weird that I'm not sure if my result is how it's supposed to be.
    • kof97pla has a few pixels swapped. It looks like incorrect vertical interlacing.
    • kof2kfd, kof2kotc, kof2kps2, kof2ksp look like they might be missing some text. kof2k2omg9 and kof2k2omg9b look like they might be missing the logo.
    • neonopon and poknight are clearly not supposed to have sound.
    • neotet, neothndr and most demos use the same V-ROM, but neopang is the only one of these to actually have sound. I'm not sure if the others are supposed to have sound, so I decided to tag only the two games among them for missing sound.
    • neotris and nsmb feel slow-paced, but I'm not sure if they are actually slower than they are supposed to be.
    • pbobblenb is included in non-merged sets, but in fact, this set is usually merged. Because the base game is supported by stock, I omitted this set.

Definitions for the "working" column (applies to madcock's list as well, except for "norun"):

  • working: The set finishes loading and there is a time after loading where you could enter the pause menu. It does not mean that the game is actually working or even playable.
  • loadcorrupt: The set finishes loading and instantly freezes the full device. This either means that the P-ROM is not as expected or that the game itself is not supported. Some of the latter cases are regressions, as sets worked in the much older MAME 2000.
  • loadhang: Loading freezes at vs-bios.rom. I believe that's because the V-ROM and its largest file (the latter is always 8 MiB) are about to exceed the 98 MiB ROM limit.
  • noload: No loading indicator is displayed in the bottom left. This is usually the case for sets that are way too big or lack a file that matches the expected metadata (names, sizes, CRCs). This cannot happen here, because I chose 32 host sets that aren't too big and all files are freshly generated by my tool, so the metadata is always correct.
  • norun: The set finishes loading but the Neo Geo runs its internal test instead of the actual game. This is usually the case because the P-ROM is not what it expects. While this falls under the definition of "working", I refused to call it like that.
